🚀 CRUIS API

EBSCO Authentication & Proxy Service

Welcome to CRUIS API

This API provides a seamless way to authenticate with EBSCO services and proxy authenticated requests.

Quick Start: Use the Authentication tab to get your auth token, then use it in the Proxy Request tab to make authenticated API calls.

Available Endpoints

  • POST /api/auth/ebsco
    Authenticate with EBSCO using library card credentials
  • GET /api/ebsco-proxy/*
    Proxy GET requests to EBSCO services with authentication
  • POST /api/ebsco-proxy/*
    Proxy POST requests to EBSCO services with authentication
  • GET /health
    Health check endpoint

EBSCO Authentication

Enter your library card credentials to obtain an authentication token.

Proxy Request

Make authenticated requests to EBSCO services using your auth token.

Enter the full path after the EBSCO domain (e.g., search.ebscohost.com/api/v1/search)

API Endpoints Reference

POST /api/auth/ebsco

Authenticate with EBSCO using library card credentials and return an authentication token.

Request Body: 
{
  "cardNumber": "1001600244772",
  "password": "your-password"
}
Response (200 OK): 
{
  "authToken": "ebsco-auth-cookie-value"
}
Error Responses:
  • 400 - Bad request (missing cardNumber or password)
  • 401 - Authentication failed
  • 500 - Server error

GET /api/ebsco-proxy/{path}

Proxy authenticated GET requests to EBSCO services.

Required Headers: 
X-Auth-Token: your-auth-token-here
Path Parameter:
  • path - The target path after EBSCO domain (e.g., search.ebscohost.com/api/v1/search)
Example: 
GET /api/ebsco-proxy/search.ebscohost.com/api/v1/search
X-Auth-Token: your-auth-token

POST /api/ebsco-proxy/{path}

Proxy authenticated POST requests to EBSCO services.

Required Headers: 
X-Auth-Token: your-auth-token-here
Content-Type: application/json
Path Parameter:
  • path - The target path after EBSCO domain
Request Body: 
Optional JSON payload for the proxied request

GET /health

Health check endpoint to verify service availability.

Response (200 OK): 
{
  "status": "ok"
}

Authentication Flow

  1. GET login page from EBSCO with custId, groupId, profId, and requestIdentifier
  2. POST to login API with library card number
  3. POST to login API with password
  4. Follow redirects to extract authentication cookie
  5. Return auth token to client
